## 7. 验收中发现并修复的问题

1. AI 提示词允许重复查询上下文，导致单样本轮次和耗时失控；现限制为最多一轮并行补充查询，下一轮必须提交三分类结论。
2. 内容评估提交工具曾错误读取不存在的 `scope.node_id`；现统一使用 `ToolScope.subject_id`。
3. 模型原先可以提交错误的版本或节点标识；现由 work item 作用域覆盖版本 ID、节点 ID，并优先使用数据库中的真实节点名称。
4. 200 条真实样本所需 token 超过旧 1,000,000 默认值；现按实测调整默认 token 与 wall-time 预算，并保留预算耗尽后的续跑能力。
