## 二、本次发现并补齐的两处规范缺口

对照里程碑索引的"禁止行为 / 验收标准"，发现两处此前未满足的硬指标，本次已全部修复：

### 缺口 1：M4 §7.9 — checkpointer 仍用 InMemorySaver
- **规范**：禁止 M4 阶段仍使用 MemorySaver，必须换成 SQLite checkpointer（"服务重启后同 thread_id 可恢复"）。
- **修复**：
  - 安装 `langgraph-checkpoint-sqlite==3.1.0`（.venv）
  - 新建 `backend/app/agents/checkpoints.py`：用 `SqliteSaver`（常驻 sqlite3 连接，文件落盘 `data/workflow_checkpoints.sqlite`）
  - `backend/app/api/workflows.py` 的 `_get_workflow_checkpointer()` 由 `create_memory_checkpointer()` 改为 `create_sqlite_checkpointer()`
  - 已验证状态可跨 invoke 持久化（满足重启可恢复）
  - `data/workflow_checkpoints.sqlite` 已加入 `.gitignore`

### 缺口 2：M5 §8.6 / §8.9 — 完全没有 SSE 实时事件流
- **规范**：前端必须消费真实 SSE 事件（`GET /api/workflows/{task_id}/events`），禁止静态假进度条。
- **修复**：
  - 新建 `backend/app/agents/events.py`：`map_workflow_event` / `interrupt_event` / `completed_event` / `format_sse`（事件类型对齐规范 §8.5）
  - `backend/app/repositories/task_repo.py` 新增 `list_events(workflow_id, after_id)`
  - `backend/app/api/workflows.py` 新增 `GET /api/workflows/{task_id}/events`（`StreamingResponse`，轮询推送 `workflow_step` / `workflow_interrupt` / `workflow_completed` / `workflow_failed`）
  - 前端 `frontend/src/api/workflows.ts` 新增 `workflowEvents(taskId)`（EventSource）
  - 新建 `frontend/src/components/TaskStatusBar.vue`：消费 SSE + 实时事件日志 + 进度条（让"真实 Agent 行为"端到端可见）
  - `frontend/src/views/WorkflowView.vue` 接入 `TaskStatusBar` 作为真实进度来源（保留轮询作兜底）
