# 1. 问题定义

本系统面向标准产品分类体系，主要存在四类问题：

------

## 1.1 结构类问题

标准产品体系在树形结构中存在以下问题：

- 层级过深（路径冗长）
- 节点过宽（子节点过多）
- 结构不均衡（不同分支差异大）
- 父节点缺失（结构断裂）

👉 本质问题：**树结构不合理**

------

## 1.2 内容类问题

节点语义与挂载关系存在以下异常：

- 节点重复（同名或语义重复）
- 重复挂载（同一对象多路径出现）
- 父子关系错误（语义不匹配）
- 父子节点同名（层级冗余）
- 同义词错误（syn_list语义偏移或冲突）

👉 本质问题：**分类语义混乱**

------

## 1.3 评价问题（系统能力问题）

当前系统缺乏统一评估机制：

- 无结构质量评分
- 无内容质量评分
- 无同义词覆盖评价
- 无优化前后对比标准

👉 本质问题：**无法量化系统效果**

------

## 1.4 版本问题（工程能力问题）

系统缺乏过程管理能力：

- 修改过程不可追踪
- 前后版本不可对比
- 无法回滚历史状态
- 无维护日志记录

👉 本质问题：**缺乏工程化管理能力**

------

# 2. 解决方案设计初版

系统采用：

> **规则检测 + 语义判断 + 评价体系 + 版本管理** 的四层结构

------

## 2.1 结构问题解决方案（Rule-based）

### 方法：

- 构建树结构（ID + parent_id）
- 计算层级深度
- 统计子节点数量
- 检测断裂节点

### 输出：

- 层级过深节点
- 节点过宽节点
- 结构不均衡区域
- 缺失父节点列表

### 策略：

- 过深 → 合并/压缩层级
- 过宽 → 拆分子类
- 断裂 → 补父节点

------

## 2.2 内容问题解决方案（Rule + LLM）

### 方法：

- 名称去重检测
- 路径一致性分析
- 多路径挂载检测
- 父子语义一致性判断（LLM辅助）
- 同义词一致性检测

### 输出：

- 重复节点
- 错误挂载节点
- 父子关系异常
- 同义词冲突列表

### 策略：

- 明确规则 → 自动修正
- 语义问题 → LLM判断 + 人工确认

------

## 2.3 评价问题解决方案（系统级指标）

### 方法：

构建三类评分体系：

#### （1）结构评分

- 最大深度
- 平均深度
- 节点宽度分布
- 不均衡度

#### （2）内容评分

- 重复率
- 错误挂载率
- 父子异常率

#### （3）语义评分

- 同义词覆盖率
- 同义词冲突率

### 输出：

- 单版本评分
- 前后版本对比
- 优化效果量化

👉 本质：**系统健康度指标**

------

## 2.4 版本问题解决方案（工程日志系统）

### 方法：

- 记录每次修改操作
- 保存修改前后节点状态
- 记录修改原因与来源
- 保存版本快照

### 支持能力：

- 版本对比（diff）
- 修改追踪（trace）
- 状态回滚（rollback）

### 输出：

- 版本变更日志
- 节点变化列表
- 修改链路记录

👉 本质：**Agent运行审计系统**

------
