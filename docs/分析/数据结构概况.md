# 数据结构概况

## 1. 数据结构分析

### 1.1 数据整体形态

当前标准产品体系数据以 Excel 表格形式提供。表中每一行代表标准产品体系中的一个分类节点。整体来看，该数据不是普通二维表，而是一个以表格形式存储的树形结构。

数据中的核心字段如下：

| 字段名 | 含义 | 作用 |
|---|---|---|
| `category_id` | 当前节点 ID | 唯一标识一个产品分类节点 |
| `category_name` | 当前节点名称 | 表示标准产品分类名称 |
| `category_group_id` | 祖先节点 ID 路径 | 用于构建树形结构 |
| `category_pids` | 父节点路径 | 包含虚拟根节点，可用于路径一致性校验 |
| `category_group_name` | 祖先节点名称路径 | 用于人工理解完整分类路径 |
| `syn_list` | 同义词列表 | 用于同义词覆盖分析和产品名称匹配 |

---

### 1.2 树结构理解

其中，`category_group_id` 是构建树结构的关键字段。

需要注意的是，`category_group_id` 不是单个父节点，而是一条从顶层节点到当前节点父节点的祖先路径。

例如：

| category_id | category_name | category_group_id | category_group_name |
|---:|---|---|---|
| 6 | 早籼稻 | 2,3,4,5 | 农、林、牧、渔业类产品,农业产品,谷物,稻谷 |

该行表示：

```text
农、林、牧、渔业类产品
  → 农业产品
    → 谷物
      → 稻谷
        → 早籼稻
```

因此，对于某个节点：

```text
直接父节点 = category_group_id 中最后一个 ID
```

如果 `category_group_id` 为空，说明该节点是顶层大类节点。

---

### 1.3 数据可以分为三层理解

#### 1. 节点层

每一行表示一个节点，包括：

- 节点 ID；
- 节点名称；
- 同义词列表。

#### 2. 结构层

通过 `category_id` 和 `category_group_id` 构建父子关系，形成产品分类树。

#### 3. 语义层

通过 `category_name` 和 `syn_list` 表示产品分类的语义信息，用于后续判断：

- 节点是否重复；
- 同义词是否覆盖充分；
- 父子关系是否合理；
- 节点是否需要合并、拆分或迁移。

### 2.1 顶层大类统计

标准产品体系中共有 12 个顶层大类，包括：

1. 农、林、牧、渔业类产品；
2. 矿产品；
3. 食品、饮料、烟、酒类产品；
4. 纺织、服装、皮革类产品；
5. 木制品、家具类产品；
6. 纸、印刷品、软件、文教、工艺品类产品；
7. 石油、化工、医药产品；
8. 金属及非金属制品；
9. 机械、设备类产品；
10. 电力、热力、水、燃气类产品；
11. 销售服务；
12. 产业链。

其中，“机械、设备类产品”节点数量最多，是当前体系中规模最大的部分，后续可以作为重点分析对象。
