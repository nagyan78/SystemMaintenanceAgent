## 算法

### 树

#### 1. 树的高度

**相对阈值判断**

超过平均高度 + 2σ 的路径，视为异常深路径。

$$
子树高度 > mean(H) + 2σ
$$

| 方法       | 解决什么     |
| ---------- | ------------ |
| mean + 2σ  | 找异常深路径 |
| 路径冗余率 | 找无意义层级 |

**解决方法**

调用 model 分析异常深路径中是否存在路径冗余，判断是否有不必要的中间层级。

#### 2. 树的宽度

**P95 阈值判断**

当某节点的子节点数量超过所有节点子节点数量的 95% 时，进入候选异常集合。

$$
k > P95
$$

| 方法    | 解决什么                 |
| ------- | ------------------------ |
| k > P95 | 找子节点数量异常多的节点 |

**相对分支爆炸比**

用于判断当前节点是否明显比兄弟节点更宽。

$$
相对分支爆炸比 = \frac{当前节点的子节点数}{mean(兄弟节点子节点数)}
$$

| 相对分支爆炸比 | 说明     |
| -------------- | -------- |
| < 2            | 正常     |
| 2–5            | 偏宽     |
| > 5            | 明显过宽 |

**解决方法**

接入 model，判断当前节点下是否直接挂载了不同语义子类，并分析这些子类之间是否可以继续分组。

#### 3. 树平衡问题

**最大/最小子树规模比值算法**

用于判断同一层级下不同分支之间的规模差异。

$$
balance_ratio = \frac{max(subtree_size)}{min(subtree_size)}
$$

| balance_ratio | 说明       |
| ------------- | ---------- |
| < 5           | 基本均衡   |
| 5–20          | 不均衡     |
| > 20          | 严重不均衡 |

**信息熵判断**

用于判断节点子树规模分布是否均匀。

- n = 当前节点的直接子节点数量

- p_i = 第 i 个子节点所占的比例

$$
H = -\sum_i p_i \log p_i
$$
$$
H_{norm} = \frac{H}{\log n}
$$

| H_norm  | 说明       |
| ------- | ---------- |
| > 0.8   | 较均衡     |
| 0.5–0.8 | 一般       |
| < 0.5   | 明显不均衡 |

#### 4. 叶子节点占比

**叶子节点占比判断**

用于判断当前树中叶子节点是否过多，进而分析是否存在可以继续细分的节点。

$$
叶子节点占比 = \frac{leaf_nodes}{total_nodes}
$$

| 叶子节点占比 | 说明                         |
| ------------ | ---------------------------- |
| 较低         | 分类层级仍有展开空间         |
| 适中         | 结构相对正常                 |
| 较高         | 可能存在大量未继续分层的节点 |

**解决方法**

对叶子节点较多的分支接入 model，判断是否存在可以继续细分的语义类别。

### 内容类问题

#### 1. 父子节点不匹配

**父子节点不匹配判断**

用于判断父节点和子节点在语义领域上是否一致，避免出现跨领域分类挂载错误。

# 例如：

```text
父节点：工业产品类 -> 工业
子节点：薯类 -> 农业
```

这类问题说明子节点“薯类”更偏向农业或食品领域，不适合挂载在“工业产品类”下面。

**判断思路**

可以让大语言模型先根据根目录或一级分类生成关键提示词，例如：

```python
{
  "农业": ["农", "粮", "谷", "米", "麦", "薯", "豆", "果", "蔬菜", "畜", "禽"],
  "工业": ["工业", "机械", "设备", "机床", "泵", "阀", "钢", "化工"],
  "食品": ["食品", "饮料", "肉", "奶", "糖", "酒", "调味"],
  "服装": ["服装", "衣", "裤", "鞋", "帽", "纺织"],
  "电子": ["电子", "手机", "电脑", "数码", "芯片", "电路"]
}
```

然后再调用规则进行关键词匹配。如果某个子树中出现明显属于其他领域的关键词，例如含有“电子”相关词的节点出现在“服饰”分类下，则可以标记为疑似父子节点不匹配。

| 判断情况                 | 说明                     |
| ------------------------ | ------------------------ |
| 父子节点关键词领域一致   | 父子关系基本合理         |
| 子节点包含其他领域关键词 | 可能存在跨领域挂载错误   |
| 父节点领域不明确         | 需要结合上下文进一步判断 |
| 规则无法确定             | 接入 model 进行复核      |

**解决方法**

先通过关键词词库进行低成本规则筛选，找出疑似跨领域的父子节点组合；再对高风险节点接入 model，判断父子节点之间是否存在合理的包含关系，并给出调整建议。

------

#### 2. 命名不规范

**命名规范性判断**

用于判断节点名称是否符合分类体系的命名要求，避免出现名称过长、过短、格式混乱、混入型号、空值等问题。

| 问题类型     | 判断方式                     | 说明                       |
| ------------ | ---------------------------- | -------------------------- |
| 名称过长     | 检查字符长度是否超过阈值     | 可能混入描述性语句         |
| 名称过短     | 检查字符长度是否过短         | 可能表达不清               |
| 特殊符号异常 | 检查是否包含大量特殊符号     | 可能影响展示和匹配         |
| 数字型号混入 | 检查是否包含型号、规格、编号 | 可能把具体商品当作分类节点 |
| 空值节点     | 检查名称是否为空             | 数据本身存在缺失           |

例如：

```text
海尔BCD-216冰箱
手机/电脑&数码类
其他
NULL
```

这些节点可能存在命名不规范问题。

**解决方法**

系统可以通过名称长度、特殊符号、数字型号、空值检测等规则进行初步筛选。对于明显异常的节点可以直接标记为问题节点；对于需要语义判断的节点，再接入 model 生成更加规范的推荐名称。

------

#### 3. 抽样调查

**抽样调查判断**

用于处理规则难以完全判断的内容类问题。由于内容类问题往往涉及语义理解，如果对 Excel 中所有节点逐条调用大语言模型，会带来较高的时间成本和算力消耗，因此需要采用抽样分析或候选节点筛选的方式。

| 问题类型     | 问题解释                                                     |
| ------------ | ------------------------------------------------------------ |
| 近义节点     | 同一体系中存在含义相近、表达不同的分类节点，容易造成重复建设或归类混乱 |
| 模糊节点     | 节点名称过于宽泛，如“其他”“综合”“通用”等，无法准确表达分类边界 |
| 父子关系倒挂 | 父节点与子节点的层级关系不合理，可能出现子类比父类更宽泛，或父子概念冲突的情况 |
| 粒度不一致   | 同一层级节点的抽象程度不统一，部分节点混入具体型号、规格、商品名称等过细信息 |

**解决方法**

由于近义节点、模糊节点、父子关系倒挂、粒度不一致等问题主要依赖语义判断，规则不一定能提前筛选出问题节点。因此，这部分内容类问题可以从一开始就采用随机抽样方式处理。

整体流程如下：

```text
全部节点
  ↓
随机抽样
  ↓
构造分析样本
  ↓
批量调用 model 判断
  ↓
生成问题说明和修改建议
```

抽样时可以根据问题类型选择不同样本：

| 问题类型     | 抽样对象               |
| ------------ | ---------------------- |
| 近义节点     | 抽取同层节点或同类节点 |
| 模糊节点     | 抽取节点名称           |
| 父子关系倒挂 | 抽取父子节点对         |
| 粒度不一致   | 抽取兄弟节点组         |

**抽样策略**

| 抽样方式         | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| 高风险节点优先   | 优先抽取被规则命中的可疑节点                                 |
| 分层抽样         | 每个一级分类或主要分支都抽取一部分节点，避免只集中在某一类   |
| 兄弟节点成组抽样 | 对同一父节点下的多个子节点一起判断，适合分析粒度和分类口径问题 |
| 深层路径抽样     | 抽取层级较深的路径，判断是否存在父子倒挂或层级冗余           |
| 大分支抽样       | 对子节点数量较多的分支进行重点分析                           |

这样可以在不全量调用大模型的情况下，发现部分隐藏的语义问题，并降低系统运行时间和算力消耗。
