# 产品标准体系诊断与自动整改解决方案

> 状态：DESIGN BASELINE
> 更新日期：2026-07-21
> 适用对象：产品标准体系 Excel 的结构诊断、内容抽样评估、自动整改、版本复诊与报告生成
> 事实边界：本文定义目标算法和验收口径；功能是否已经实现，仍以当前代码、自动化测试和“开发文档/00_当前状态”中的事实记录为准。

## 1. 目标与产品边界

系统面向包含数万节点的产品分类体系，完成以下闭环：

1. 导入并保存原始 Excel，不覆盖源文件；
2. 对全部节点执行确定性结构诊断；
3. 对全部节点执行低成本内容规则诊断；
4. 对 200 个分层随机样本执行 AI 内容合理性评估；
5. 计算结构、内容规则、AI 内容抽样和综合质量评分；
6. 为确认的问题匹配固定整改策略；
7. 由 AI 填写策略参数，并由独立 AI 二次复核；
8. 通过确定性校验和完整快照预演后自动执行；
9. 生成不可变新版本，复诊并生成可追溯报告。

系统不设置人工审核环节。LLM 不直接修改 Excel、SQLite 或 Qdrant，只能生成结构化判断和整改参数；所有副作用由确定性服务执行。

现有数据库中的 `review_batch` 继续作为“AI 复核批次”保存方案集合、独立复核结论和执行证据，不再提供用户批准、拒绝或编辑入口。保留表名仅用于兼容既有数据和减少提交前迁移风险，不改变其领域含义。

## 2. 模型与基础能力

| 能力 | 实现方式 | 是否必须 |
|---|---|---|
| Excel 解析、树构建、统计计算 | Python、SQLite、确定性规则 | 必须 |
| 常规语义判断 | DeepSeek API | AI 内容抽样时必须 |
| AI 修改方案生成 | DeepSeek API | 自动整改时必须 |
| 独立 AI 二次复核 | 独立模型调用、独立提示词与输入快照 | 自动整改时必须 |
| Embedding 与相似节点检索 | DashScope Embedding、Qdrant | 可选增强 |
| 动作校验与快照预演 | 确定性程序 | 必须 |

Embedding 与 Qdrant 可辅助 AI 查找相似节点，但不作为 200 个分层随机样本能否生成的前置条件。外部模型不可用时，系统保留全量规则结果并生成部分完成报告。

## 3. 输入数据与导入边界

### 3.1 必需字段

Excel 第一张工作表必须包含：

- category_id
- category_name
- category_group_id
- category_pids
- category_group_name
- syn_list

输入数据域仅包含以上六个字段。父节点、节点层级、叶子状态、标准化路径和诊断状态均为系统派生信息，不是新的 Excel 源字段。

路径字段的输入与输出格式固定为：

- category_group_id：按根到直接父节点顺序使用逗号分隔祖先 ID，例如 2,3,4；
- category_pids：保留 [-1] 根标记，每个祖先 ID 使用方括号包裹，例如 [-1],[2],[3],[4]；
- category_group_name：按根到直接父节点顺序使用逗号分隔祖先名称；
- 根节点的 category_group_id 和 category_group_name 为空，category_pids 为 [-1],。

### 3.2 导入阻断错误

以下问题会导致分类体系无法可靠建立，必须拒绝导入：

- 缺少必需字段；
- category_id 为空或无法解析为整数；
- category_id 重复；
- category_name 为空；
- Excel 文件损坏或无法读取。

### 3.3 可诊断结构缺陷

以下问题不阻断导入，应保存原始数据并在诊断阶段报告：

- 直接父节点缺失；
- 祖先链包含不存在的节点；
- 层级异常深；
- 节点异常宽；
- ID 路径与名称路径不一致；
- 同义词字段格式异常；
- 分类内容或语义可能不合理。

## 4. 总体执行流程

    上传 Excel
      → 校验导入阻断错误
      → 解析并保存初始版本
      → 全量结构诊断
      → 全量内容规则诊断
      → 200 个分层随机样本
      → AI 内容判断与一次证据补全重试
      → 计算三个子分和综合质量评分
      → 为确认问题匹配固定整改策略
      → AI 生成完整策略参数
      → 独立 AI 二次复核
      → 确定性校验
      → 完整快照预演
      → 自动执行并生成新版本
      → 新版本复诊、重新评分和报告

规则诊断覆盖全部节点，不调用 LLM。AI 内容评估只处理固定的分层随机样本，不宣称逐个判断全部节点。

## 5. 全量结构诊断

### 5.1 父节点缺失

#### 判断方法

category_group_id 中最后一个祖先 ID 是节点的直接父节点。若该 ID 不在当前版本的 category_id 集合中，则记录“父节点缺失”。

同时检查完整祖先链，统计受同一缺失祖先影响的全部节点。

#### 固定整改策略

1. 如果路径字段同时提供缺失父节点的 ID 和名称，按原 ID、名称及最近有效祖先补建父节点；
2. 多个节点引用同一缺失父节点时，只补建一次；
3. 如果名称无法可靠取得，由 AI 根据路径和子节点推断名称，并进行独立二次复核；
4. 如果仍无法确定名称，将受影响节点连接到最近有效祖先，不创建虚构名称；
5. 预演必须确认不存在重复 ID、循环、断链和路径不一致。

### 5.2 异常深路径

#### 指标定义

- 节点深度：根节点到当前节点的节点数量；
- 路径长度：根节点到叶子节点的一条完整路径包含的节点数量；
- 子树高度：当前节点到其最深叶子节点的最大距离。

层级过深使用固定路径长度判断，根节点计为第一层：

    maximum_path_length = 7

路径长度大于 7 时直接形成“路径过深问题”，进入结构问题计数和结构评分。

每条超过阈值的根到叶路径记录一个问题，问题主体为该路径的叶节点，并保存完整路径、实际长度、所属根分类、阈值和需要减少的层数。同一条路径中的多个深层节点不得重复生成问题。

风险等级按需要减少的层数确定：

| 需要减少的层数 | 风险等级 | 结构评分权重 |
|---:|---|---:|
| 1 | 低 | 0.2 |
| 2 | 中 | 0.6 |
| 3 及以上 | 高 | 1.0 |

#### 固定整改策略

需要删除的层数按以下公式计算：

    remove_count = current_path_length - 7

AI 不再判断路径是否过深，只从路径中选择 remove_count 个待删除的中间节点。选择时优先考虑：

- 只有一个直接子节点；
- 没有独立同义词；
- 删除后父子语义仍然合理；
- 删除后不会造成同级重名或新的异常宽节点。

根节点和叶子节点禁止删除。

多子节点中间层允许删除，但必须同时满足：

- 全部直接子节点能够完整改挂到原父节点；
- 改挂后不产生同级重名；
- 原父节点不会因此形成新的异常宽节点；
- 独立 AI 复核确认不会丢失必要的分类边界。

异常路径按当前路径长度从深到浅处理。每完成一组删除动作，都必须在工作快照中重新计算全部路径长度；已经恢复到阈值以内的共享路径自动移出待处理集合，避免对同一中间节点生成重复删除动作。

执行步骤：

1. AI 输出待删除中间节点 ID 列表；
2. 独立 AI 复核节点组合和删除后父子语义；
3. 将每个待删除节点的直接子节点连接到其父节点；
4. 保留其余子树内部结构；
5. 从新版本中物理删除选定中间节点；
6. 递归更新所有后代节点的 category_group_id、category_pids 和 category_group_name；
7. 所有保留节点的 category_id、category_name 和 syn_list 保持不变；
8. 在执行记录中保存被删除节点的完整内容、原父子关系、删除原因和子节点改挂结果；
9. 校验不存在循环、断链、重名冲突、异常宽度恶化和深度未改善。

物理删除只发生在新版本。原始 Excel 和旧版本保持不可变，回滚通过切换到旧版本完成。

如果没有足够的中间节点通过安全校验，只执行能够通过校验的删除动作；未达到阈值的剩余部分继续保留为有效问题并影响评分。

### 5.3 异常宽节点

#### 判断方法

只统计非叶子节点的直接子节点数量，使用固定阈值：

    maximum_direct_children = 80

直接子节点数量大于 80 时直接形成“节点过宽问题”，进入结构问题计数和结构评分。后代节点总数不参与该阈值。

风险等级直接按满足每组最多八十个节点时所需的最少分组数确定：

| 直接子节点数 | 最少分组数 | 风险等级 | 结构评分权重 |
|---:|---:|---|---:|
| 81～160 | 2 | 低 | 0.2 |
| 161～240 | 3 | 中 | 0.6 |
| 241 及以上 | 4 及以上 | 高 | 1.0 |

#### 固定整改策略

节点一旦超过固定宽度阈值就必须应用“新增中间分组”策略。AI 不判断是否需要整改，只负责生成语义分组方案：

1. 最少分组数为 ceil(直接子节点数 ÷ 80)，AI 可以根据语义增加分组数；
2. AI 只输出每个新分组的名称、成员节点和分类依据，不生成 category_id 或路径字段；
3. 每个原子节点必须恰好进入一个分组，不得遗漏或重复；
4. 每个分组至少包含两个节点且不超过八十个直接子节点；
5. 不允许产生空分组；
6. 新分组名称不得与现有同级名称冲突；
7. 各分组必须具有明确且互不重叠的分类含义；
8. 预演全部移动动作并递归更新后代路径；
9. 分组后重新计算宽度、深度和语义一致性。

新中间分类的 category_id 由程序确定性生成：先按规范化后的分组名称排序，名称相同时按分组内最小成员 category_id 排序，再从当前版本最大 category_id 加一开始顺序分配。category_group_id、category_pids 和 category_group_name 同样由程序依据整改后的树结构生成。相同输入与相同分组方案必须得到完全相同的新 ID 和路径字段。

新增中间分类仍须输出完整六列：category_name 由 AI 生成并经独立二次复核，syn_list 固定为 `[]`；category_id、category_group_id、category_pids 和 category_group_name 均由程序生成。AI 不得为新增分类编造同义词、分类 ID 或路径。

每次整改最多允许一个兜底分组。兜底分组名称必须包含父节点范围，例如“其他煤化工设备与试剂”，不得只使用“其他”；兜底分组不得超过八十个节点，也不能成为成员数量最多的分组。能够归入明确语义组的节点禁止放入兜底分组。

全部分组通过独立 AI 复核、确定性校验和快照预演后自动执行。

#### 结构整改执行顺序

结构整改采用固定的串行顺序，不设计宽度与深度的联合规划事务：

1. 先完成当前版本中的全部节点过宽整改；
2. 再以宽度整改后的完整结果树重新检测路径深度；
3. 所有超过七层的路径使用既定路径压缩规则处理；
4. 本轮新增的宽度分组节点不进入路径压缩删除候选集；
5. 最后统一验证全部节点的直接子节点数不超过八十、全部根到叶路径不超过七层；
6. 两项结构约束均通过后才允许输出新版本，否则保留失败证据且不发布该整改版本。

### 5.4 树平衡指标

树平衡只作为解释性证据，不直接扣分或执行修改。

对子节点数大于 1 的节点计算：

    balance_ratio = max(subtree_size) / max(min(subtree_size), 1)

归一化信息熵：

    p_i = subtree_size_i / sum(subtree_size)
    H = -sum(p_i × log(p_i))
    H_norm = H / log(n)

当 n 小于等于 1 时不计算 H_norm。

| 指标 | 参考解释 |
|---|---|
| balance_ratio 小于 5 | 基本均衡 |
| balance_ratio 为 5 至 20 | 不均衡候选 |
| balance_ratio 大于 20 | 严重不均衡候选 |
| H_norm 大于 0.8 | 较均衡 |
| H_norm 为 0.5 至 0.8 | 一般 |
| H_norm 小于 0.5 | 明显不均衡候选 |

业务分支天然可能不均衡，因此必须结合语义和使用场景解释。

### 5.5 叶子节点占比

    leaf_ratio = leaf_node_count / total_node_count

叶子节点占比只作为体系概览和版本对比指标，不直接认定问题、不直接扣分、不自动细分。只有 AI 在抽样中发现明确的粒度不一致或缺失分类证据时，才创建正式问题。

## 6. 全量内容规则诊断

内容规则检查全部节点，只处理可以确定判断的数据规范问题，不承担业务语义判断。

### 6.1 名称与字段规则

- category_name 为空：导入阻断错误；
- 名称首尾存在异常空白：名称格式错误；
- 名称包含异常换行或不可见字符：格式问题；
- ID 路径与名称路径长度不一致：路径字段不一致；
- category_pids 去除根哨兵后与 category_group_id 不一致：路径字段不一致；
- 同一父节点下名称完全相同：同级重复问题。

名称过长、过短、包含数字或连接符只能形成 AI 判断候选，不能仅凭字符命中直接改名。代码不得针对具体产品名称写死诊断结果。

固定风险和整改规则如下：

| 问题 | 风险等级 | 权重 | 固定整改方式 |
|---|---|---:|---|
| 路径字段不一致 | 高 | 1.0 | 依据整改后的实际树关系重新生成三个路径字段 |
| 同级名称完全重复 | 高 | 1.0 | AI 在“合并”或“重命名”策略中选择并填写参数，独立复核后执行 |
| 异常换行或不可见字符 | 中 | 0.6 | 确定性删除不可见字符并将换行规范为空格 |
| 名称首尾异常空白 | 低 | 0.2 | 确定性去除首尾空白 |

### 6.2 同义词规则

同义词列表执行以下确定性规范化：

1. 解析列表；
2. 删除 null、空字符串和异常空白；
3. 删除与节点主名称相同的词；
4. 按忽略大小写的方式去重；
5. 统一中英文标点和列表格式；
6. 保持其余同义词原有顺序。

解析失败、空值、自引用和重复值属于“同义词格式错误”。某个词是否真正等价、范围是否过宽，交给 AI 判断。

同义词字段无法解析为列表属于中风险问题，权重 0.6；null、空字符串、异常空白、自引用、重复值和标点格式问题属于低风险问题，权重 0.2。确定性规范化必须保留原始字段值作为证据，不调用 LLM，也不改变剩余词的相对顺序。

### 6.3 内容规则评分

内容规则评分只反映全量文本和字段卫生，在综合质量评分中占 10%。同一节点命中多个内容规则时，只按最严重的问题扣分一次。

## 7. AI 内容分层随机抽样

### 7.1 抽样目的

内容语义无法通过规则全量准确判断。系统使用分层随机样本估算整体内容质量，同时保存样本中发现的正式问题。

风险候选诊断与分层随机抽样不能混为一谈。用于评分的样本不得优先选择可疑节点，否则问题率会被系统性高估。

### 7.2 样本量与分层字段

- 总样本量：200；
- 抽样方式：不放回抽样；
- 联合分层键：根分类、节点层级、叶子或非叶子、同义词有或无；
- 分配方式：先保证每个非空根分类至少一个样本，剩余额度按联合分层节点数比例使用最大余数法分配；样本额度不足时不强制覆盖每个联合小层；
- 随机种子：由诊断运行记录并固定，默认使用项目配置值；
- 可复现性：同一版本和同一随机种子必须得到相同样本。

抽样前按联合分层键和 category_id 稳定排序，再使用固定随机种子在各层内抽取。若有效节点总数不足 200，则抽取全部有效节点，不重复补足。报告保存每个联合分层的总体数量、分配额度和实际样本数。

### 7.3 AI 判断上下文

每个样本至少提供：

- 节点名称和完整路径；
- 父节点；
- 同级节点代表项；
- 直接子节点代表项；
- 同义词；
- 节点层级和叶子属性。

Embedding 可用于补充相似节点，但不是必需输入。

### 7.4 判断标准

AI 统一检查：

1. 节点名称是否清晰并符合产品分类表达；
2. 节点与父节点是否具有合理的上下位关系；
3. 节点与同级分类是否采用一致的分类维度；
4. 同义词是否与主名称等价，没有扩大或缩小语义范围。

### 7.5 二分类结论

| 结论 | 定义 | 样本得分 |
|---|---|---:|
| 合理 | 基于现有分类树证据未发现明确问题 | 1 |
| 存在问题 | 存在明确的命名、层级、维度或同义词问题 | 0 |

AI 判定“存在问题”时必须同时提交具体问题类型、问题证据和原因，系统随后按照固定整改规则生成方案并进入独立 AI 复核。模型调用失败或在限定轮次内未提交有效二分类结论时，该样本记为未完成，不进入评分，也不伪造合理或问题结论。

### 7.6 AI 内容抽样评分

    AI_content_sample_score =
      100 × reasonable_count / (reasonable_count + problem_count)

报告必须同时展示合理、不合理、未完成数量、样本节点、随机种子和实际完成数，不能只展示单一分数。

## 8. 三项子分与综合评分

### 8.1 结构评分

结构评分基于全量结构问题，使用每千节点风险扣分制。每个受影响节点只按最严重的有效问题扣分一次：

- 高风险问题：10 个扣分点；
- 中风险问题：5 个扣分点；
- 低风险问题：1 个扣分点。

    structure_score =
      max(0,
        100 - 1000 / max(total_nodes, 1000)
              × (10 × high_nodes + 5 × medium_nodes + low_nodes)
      )

分数下限为 0。小于 1000 个节点的数据按 1000 个节点归一化，避免小样本中的单个问题支配全部分数。存在未解决结构问题时，不得因四舍五入显示为 100。

### 8.2 内容规则评分

    content_rule_score =
      max(0,
        100 - 1000 / max(total_nodes, 1000)
              × (10 × high_nodes + 5 × medium_nodes + low_nodes)
      )

扣分规则与结构评分一致，同一节点只按最严重问题扣分一次。该公式衡量问题密度和风险强度，不设置目标分、封顶分，也不针对具体文件写入特殊规则。

### 8.3 综合质量评分

    overall_quality_score =
      0.40 × structure_score
      + 0.10 × content_rule_score
      + 0.50 × AI_content_sample_score

报告必须同时展示三个子分和综合分。综合分不是 AI 准确率，也不是诊断覆盖率。

若 AI 内容抽样因外部模型不可用而未完成，则不伪造综合分，报告状态为部分完成，并保留结构评分和内容规则评分。

## 9. 问题类型与固定整改策略

固定整改策略限定动作类型、必填参数、禁止条件、影响范围、校验和复诊规则。AI 只能填写策略允许的目标值。

| 问题类型 | 默认动作 | 核心约束 |
|---|---|---|
| 父节点缺失 | 补建父节点或连接最近有效祖先 | ID 唯一、名称有证据、无循环 |
| 异常深路径 | 删除规定数量的中间节点并改挂子树 | 根、叶及本轮新增宽度分组禁止删除 |
| 异常宽节点 | 新增中间分组并移动子树 | 分组互斥、名称不冲突 |
| 同级重复 | 合并或重命名 | 保留节点明确、完整迁移后代 |
| 父子命名重复 | 合并、重命名或折叠层级 | 不破坏独立业务边界 |
| 父子语义不匹配 | 移动子树 | 目标父节点语义匹配、无循环 |
| 分类维度不一致 | 重新分组或移动 | 同级维度一致 |
| 同义词格式错误 | 确定性规范化 | 去空、去重、去主名称 |
| 同义词拼写错误 | 替换规范词 | 保留原词证据 |
| 同义词语义冲突 | 删除冲突词 | 必须有不等价证据 |
| 父子同义词重叠 | 从范围不当节点删除重叠词 | 不删除必要主名称 |
| 命名不规范 | 重命名 | 新名称符合父级范围和同级口径 |
| 语义重复 | 合并节点 | 保留节点明确、完整迁移后代与同义词 |
| 未知问题 | 先重新分类 | 未明确类型前禁止执行 |

所有确认存在问题的样本都必须尝试生成整改方案。方案不完整时自动补充证据并重新生成，最多两次；仍无法形成安全动作时保留为未整改问题，不伪造完成。

## 10. AI 自动复核与安全执行

### 10.1 独立 AI 二次复核

复核调用必须与方案生成调用分离，并检查：

- 问题证据是否充分；
- 问题类型是否正确；
- 整改策略是否匹配；
- 目标名称或目标父节点是否有依据；
- 影响范围是否完整；
- 是否存在更低风险方案。

复核不通过时重新生成方案，最多两次。

### 10.2 确定性校验

至少检查：

- 动作类型属于该问题的固定整改策略；
- 必填参数完整；
- 节点和目标节点存在；
- 不产生重复 ID；
- 不产生循环或断链；
- 不移动到自身或后代；
- 不覆盖原始 Excel；
- 不执行空动作或越权动作；
- 同一动作具备幂等键。

### 10.3 完整快照预演

在副本中应用整个动作批次并重新计算：

- 节点总数和根节点数；
- 父子关系；
- 全部后代路径；
- 重复名称；
- 缺失父节点；
- 最大深度和宽度；
- 三项子分和综合分。

只有完整批次预演通过才允许执行。执行失败时保留 workflow、run、动作和错误证据，不创建伪成功版本。

### 10.4 版本与复诊

- 原始 Excel 和既有版本不可变；
- 自动整改生成新版本；
- 新版本 Excel 严格保持 category_id、category_name、category_group_id、category_pids、category_group_name、syn_list 六列，不增加系统内部字段；
- 评分、问题、复核、执行和诊断状态只保存在数据库与报告中；
- 新版本重新执行全量结构规则和全量内容规则；
- AI 抽样使用记录的抽样策略和随机种子；
- 无论质量升降都保留新版本和证据；
- 回滚由用户主动触发。

## 11. 报告与可追溯证据

报告必须包含：

- 输入文件、输入版本、输出版本；
- workflow ID、run ID 和模型信息；
- 总节点数、规则扫描节点数；
- 分层方式、样本量、随机种子和实际完成数；
- 合理、存在问题、证据不足的样本数量；
- 结构评分、内容规则评分、AI 内容抽样评分和综合质量评分；
- 各问题类型数量、节点、路径、证据和来源；
- 自动整改方案、二次复核、校验、预演和执行结果；
- 未整改问题及原因；
- 外部依赖不可用、预算耗尽和部分完成原因。

不得展示原始 chain-of-thought，只展示决策摘要、工具、证据和动作结果。

## 12. 真实文件验收标准

以“产品标准体系.xlsx”为真实规模验收文件，当前基线为 21,090 个数据节点。

验收至少满足：

1. 文件成功导入且节点数量为 21,090；
2. category_id 全部唯一；
3. 全部 21,090 个节点完成结构规则扫描；
4. 全部 21,090 个节点完成内容规则扫描；
5. 父节点缺失作为可诊断结构缺陷，不阻断导入；
6. 按约定分层抽取 200 个不重复样本；
7. 同一版本和随机种子得到相同样本；
8. AI 样本结论只使用合理、存在问题、证据不足三类；
9. 三个子分和综合分可从事实数据重新计算；
10. 业务代码不包含针对具体产品名称写死的问题答案；
11. 修改只生成新版本，不覆盖原始 Excel；
12. 报告能够追溯节点、问题、证据、整改策略、复核、预演和执行结果。

## 13. 非目标与后续增强

当前不要求：

- 对全部 21,090 个节点逐个调用 LLM；
- 建设分布式多 Agent 平台；
- 通过人工审核授权执行；
- 使用固定产品名称模拟语义诊断；
- 在外部模型不可用时伪造 AI 结论。

后续可增强：

- 使用 Qdrant 补充相似节点证据；
- 根据多轮真实运行校准结构阈值和评分权重；
- 扩充独立 Golden 数据集以评估 AI 判断准确率；
- 增加跨版本固定样本对比和置信区间。
