# 采用全自动、原子化的分类体系维护执行

产品分类体系维护工作流不再以人工审核作为新版本生成的前置条件。智能体只可生成 `clean_synonym`、`rename_node`、`move_node`、`add_node` 和 `mark_as_valid` 五类可执行维护动作，并须先对建议完成规则校验；建议状态采用 `pending → validated → executed`，或 `pending → skipped / failed`，而非 `approved`。自动执行资格要求置信度不低于 `0.80` 且风险等级为 `low` 或 `medium`；高风险或低置信度建议标记为 `skipped`，不执行，也不阻塞同轮合格建议。只要至少一项合格建议通过校验，就将这些合格建议作为一个原子执行集合来生成新版本；没有合格建议时工作流成功完成且不创建版本，执行集合中任一动作校验或执行失败时不生成新版本并标记为失败。报告提供只读的自动决策摘要，列出建议的动作、置信度、状态及跳过或失败原因，不提供人工批准、拒绝或编辑操作。审核专用 API、页面、路由、状态字段和 LangGraph 等待审核节点将全部删除，不提供兼容入口；建议查询、版本和操作日志保留为可追溯的治理措施。现有本地运行数据仅为 mock，可删除并按新结构重建，不迁移旧审核批次或审核状态。
