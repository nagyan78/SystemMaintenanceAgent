> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` task-by-task. Every production behavior follows RED → GREEN → REFACTOR. Each task ends with one focused commit.

**Goal:** 支持从指定/最新版本继续维护；新版本生成后完成结果索引、同版质量评价、对称范围验证和人工决定是否继续下一轮。

**Architecture:** 保留现有确定性 LangGraph 主干和内容诊断/建议 Agent。阶段一增加 `import/maintain/verify` 三模式、analysis run 证据作用域、不可变版本 lineage、冻结的 `quality-v1`、Verification 和判别式 resume 契约。阶段一结果版本索引允许完整重建兜底，不宣称已经具备 outbox 增量索引。

**Tech Stack:** FastAPI、Pydantic、LangGraph、LangChain、SQLite、Qdrant、Vue 3、pytest、Node contract tests。

## 0. 冻结契约

### 0.1 三条工作流路径

```text
import:
resolve → parse → build → save v1.0 → index → structure diagnosis
→ baseline evaluation → PLAN → content diagnosis → suggestion/review/action
→ [有动作时保存结果版本并进入 post-change；无动作时报告]

maintain:
resolve/load base → create analysis run → ensure base index → structure diagnosis
→ baseline evaluation → PLAN → content diagnosis → suggestion/review/action
→ save result → result index → result evaluation → verification
→ continue interrupt / report / manual intervention

verify:
resolve base+result → ensure both indexes → evaluate base+result with quality-v1
→ symmetric verification → report / manual intervention
```

verify 不运行诊断建议、不进入审核、不执行动作。没有版本 diff 且未提供 `affected_node_ids` 时拒绝启动。

### 0.2 Analysis run 与 evidence scope

- 每个 workflow round 有一个 `analysis_run`，唯一键 `(workflow_id, round)`。
- issue、suggestion、operation log 均持久化 `workflow_id/analysis_run_id`。
- 结构诊断只替换当前 run 当前 detector 的输出，不按 version 删除历史问题。
- 报告只读取当前 workflow/run；Verification 使用 fingerprint，不使用数据库行 ID 作为跨版本身份。

### 0.3 Resume 联合契约

```python
class HumanReviewResume(BaseModel):
    interrupt_type: Literal["human_review"]
    interrupt_id: str
    decision: Literal["approve", "reject", "edit"]
    approved_suggestion_ids: list[int] = Field(default_factory=list)
    rejected_suggestion_ids: list[int] = Field(default_factory=list)
    edits: list[dict[str, Any]] = Field(default_factory=list)
    operator: str = "local_user"

class ContinueResume(BaseModel):
    interrupt_type: Literal["continue_optimization"]
    interrupt_id: str
    decision: Literal["continue", "finish"]
    operator: str = "local_user"

ResumeWorkflowRequest = Annotated[
    HumanReviewResume | ContinueResume,
    Field(discriminator="interrupt_type"),
]
```

任务状态增加 `waiting_continue`、`waiting_manual_intervention`、`completed_degraded`。请求类型、task 状态和当前 interrupt payload 不一致返回 409。`interrupt_id` 已消费时返回上次结果，不重复恢复。

### 0.4 Quality v1

`quality-v1` 只消费可针对任意不可变版本重算的确定性事实：

| 维度 | 满分 | 指标 |
|---|---:|---|
| structural_integrity | 25 | missing parent、cycle、orphan、path inconsistency 比例 |
| hierarchy_balance | 20 | deep/wide node 比例和分布 |
| semantic_consistency | 20 | 向量索引可用时的父子异常率 |
| redundancy | 15 | 同级同名率；向量可用时的语义重复率 |
| naming_quality | 10 | 名称/同义词确定性规范规则 |
| coverage_confidence | 10 | 实际 detector coverage |

缺失语义数据时维度标记 `available=false`，得分为 0，不伪造满分、不重新归一化。评价持久化 `available_points`、`coverage_ratio`、detector versions 和各维度 availability。自动通过要求 before/after 的 `score_version` 与可用维度集合一致。

## 阶段完成定义

1. v1.1 可被明确选为 base 并生成 v1.2，历史版本快照不变。
2. 同一版本多次分析不会覆盖或串用其他 workflow/run 的证据。
3. verify-only 不进入建议、审核或执行节点。
4. before/after 使用冻结的 `quality-v1`，缺失维度显式可见。
5. Verification 在同一对称范围用稳定 fingerprint 比较。
6. 用户可以通过判别式 resume 继续或结束，重复请求不重复执行。
7. required node 失败不会被报告覆盖为 completed。
8. 后端完整测试、前端 contract test 和生产 build 全部通过。
