# 阶段三：产品动作补齐 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现题目要求的完整结构治理和冗余节点治理，使过宽子树拆分、节点合并、弃用和叶子删除能够被 Agent 规划、程序模拟、人工审核并安全执行。

**Architecture:** Structure Remediation Agent 只生成结构化动作；ActionSimulationService 在不可变快照上模拟动作并生成 diff；ValidationService 负责单动作和组合冲突校验；ActionService 在审核后事务化执行并保存新版本。历史版本不修改，删除只影响新版本快照。

**Tech Stack:** Pydantic、LangChain tools、SQLite、现有 TaxonomyNodeRecord/ActionService/VersionService、Vue diff UI、pytest。

---

## 0. 前置条件与动作语义

前置：阶段二的 run/work item、候选证据、事件和并发能力稳定。

动作语义：

- `split_subtree`：在目标节点下创建多个中间节点，把原直接子节点完整且唯一地分配到新分组。
- `merge_node`：把 source 的子节点和同义词迁移到 target，在新版本中移除 source。
- `deprecate_node`：保留节点和历史引用，但在新版本中标记 `node_status=deprecated`，默认查询不展示。正式版本必须满足“任意 active 节点的祖先链全部 active”；目标仍有 active children 时必须先 move children，或显式使用 `cascade_subtree` 把完整子树一起弃用。
- `delete_leaf_node`：只允许删除无子节点、无外部引用的叶子节点；历史版本不受影响。
- 非叶子冗余节点不得直接删除，必须 merge 或 deprecate。

class SplitGroup(BaseModel):
    name: str = Field(min_length=1)
    child_ids: list[int] = Field(min_length=1)

class SplitSubtreePayload(BaseModel):
    groups: list[SplitGroup] = Field(min_length=2)

    @model_validator(mode="after")
    def unique_assignment(self) -> "SplitSubtreePayload":
        assigned = [child for group in self.groups for child in group.child_ids]
        if len(assigned) != len(set(assigned)):
            raise ValueError("split child_ids must be unique across groups")
        names = [group.name.strip() for group in self.groups]
        if len(names) != len(set(names)):
            raise ValueError("split group names must be unique")
        return self

class MergeNodePayload(BaseModel):
    source_node_id: int
    target_node_id: int
    child_strategy: Literal["move_to_target"] = "move_to_target"
    synonym_strategy: Literal["union"] = "union"

class DeprecateNodePayload(BaseModel):
    target_node_id: int
    reason: str = Field(min_length=1)
    child_strategy: Literal["require_empty", "cascade_subtree"] = "require_empty"

class DeleteLeafPayload(BaseModel):
    target_node_id: int
```

class SnapshotActionApplier:
    def apply(
        self,
        nodes: dict[int, TaxonomyNodeRecord],
        suggestion: SuggestionRecord,
    ) -> dict[int, TaxonomyNodeRecord]:
        handlers = {
            "clean_synonym": self._clean_synonym,
            "rename_node": self._rename_node,
            "move_node": self._move_node,
            "mark_as_valid": self._mark_as_valid,
            "add_node": self._add_node,
            "split_subtree": self._split_subtree,
            "merge_node": self._merge_node,
            "deprecate_node": self._deprecate_node,
            "delete_leaf_node": self._delete_leaf_node,
        }
        handler = handlers.get(suggestion.action_type)
        if handler is None:
            raise ValueError(f"Unsupported action: {suggestion.action_type}")
        handler(nodes, suggestion)
        return nodes
```

- [ ] **Step 3: 实现 ActionSimulationService**

```python
class ActionSimulationService:
    def simulate(self, version_id: int, suggestions: list[SuggestionRecord]) -> ActionPreview:
        source = TaxonomyRepository(self.settings).list_node_records(version_id, include_deprecated=True)
        nodes = {item.category_id: item.model_copy(deep=True) for item in source}
        errors: list[dict] = []
        for suggestion in suggestions:
            try:
                self.applier.apply(nodes, suggestion)
            except ValueError as exc:
                errors.append({"suggestion_id": suggestion.id, "reason": str(exc)})
        recalculated = recalculate_tree(list(nodes.values()))
        diff = VersionDiffService.compare_records(source, recalculated)
        return ActionPreview(valid=not errors, errors=errors, diff=diff, nodes=recalculated)
```

- [ ] **Step 4: 增加组合冲突检测**

同一节点被 delete/deprecate 后不能再 move/rename；两个 split 不能作用于同一目标；merge source 不能也是另一个 merge target。组合动作按依赖拓扑执行：add/split/merge/move/rename/clean 先于 deprecate/delete，使“先迁移 children、再弃用 parent”可表达。动作应用后统一检测环、重复 ID、同级重名、悬空父节点，以及“active 节点不得拥有 deprecated 祖先”的状态不变量。

- [ ] **Step 2: 实现 merge**

校验 source/target 存在且不同；target 不能位于 source 子树；把 source 直接子节点移动到 target；同义词按原顺序去重 union；从新快照删除 source；写 merge 专项 diff。

- [ ] **Step 3: 实现 deprecate**

`require_empty` 策略在组合动作完成 prerequisite move 后检查目标没有 active children，否则校验失败并阻止版本生成；`cascade_subtree` 策略把目标及全部 active descendants 标记 deprecated，并在 preview 中列出完整 affected IDs。默认树查询排除 deprecated，但管理查询、diff、rollback、export 可配置包含。版本保存前再次遍历 active 节点祖先链，发现 deprecated ancestor 立即失败，不能只在报告中提示风险。

- [ ] **Step 4: 实现 delete leaf**

只允许 active leaf、无 category 引用和未被其他本批动作作为 parent/target 的节点；从新快照删除。禁止物理删除旧版本数据库行。
