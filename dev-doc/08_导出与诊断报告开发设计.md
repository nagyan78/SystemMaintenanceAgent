# 导出与诊断报告开发设计

> 功能编号：F08  
> 独立测试目标：支持导出任意版本 Excel，并根据诊断、建议和版本数据生成 Markdown 诊断报告。  
> 相关源需求：PRD 8.9，技术架构 11.5、13、16。

---

## 1. 功能目标

为课程演示和业务复核提供可交付结果：导出指定版本的 Excel 文件，生成包含文件信息、体系统计、结构问题、内容问题、维护建议、版本变更和后续优化建议的 Markdown 诊断报告。

---

## 2. 功能边界

### 2.1 输入

1. `version_id`
2. 报告格式：
   - `markdown`
   - 可选 `pdf`
3. 导出类型：
   - Excel
   - 诊断报告

### 2.2 输出

1. `data/exports/{version_no}_taxonomy.xlsx`
2. `data/reports/{version_no}_diagnosis_report.md`
3. 下载 URL 或本地路径。

### 2.3 不包含

1. 不修改版本节点。
2. 不重新运行诊断。
3. 不执行建议。

## 4. Excel 导出设计

### 4.1 导出字段

| 字段 | 来源 |
|---|---|
| `category_id` | `category_node.category_id` |
| `category_name` | `category_node.category_name` |
| `category_group_id` | `category_node.category_group_id` 或由 `path_ids` 去掉当前节点生成 |
| `category_pids` | `category_node.category_pids` 或根据路径重建 |
| `category_group_name` | `category_node.category_group_name` 或由 `path_names` 去掉当前节点生成 |
| `syn_list` | `category_node.syn_list` |

导出排序：

1. 优先按 `path_ids` 的树路径排序。
2. 同级节点按原始导入顺序或 `category_id` 排序。

### 4.2 导出接口

```text
GET /api/versions/{version_id}/export
```

响应：

```json
{
  "version_id": 2,
  "version_no": "v1.1",
  "file_name": "v1.1_taxonomy.xlsx",
  "download_url": "/api/downloads/exports/v1.1_taxonomy.xlsx"
}
```

## 5. 报告内容结构

Markdown 报告必须包含：

1. 报告标题和版本信息。
2. 文件基本信息。
3. 分类树统计信息。
4. 结构问题统计。
5. 内容问题统计。
6. 典型问题案例。
7. 智能体维护建议摘要。
8. 已执行动作和版本变更记录。
9. 质量评分。
10. 后续优化建议。

报告模板：

```markdown
# 产品标准体系诊断报告

## 1. 基本信息

## 2. 体系统计

## 3. 结构诊断结果

## 4. 内容诊断结果

## 5. 典型问题案例

## 6. 智能维护建议

## 7. 版本变更记录

## 8. 质量评分

## 9. 后续优化建议
```

## 6. API 设计

### 6.1 生成报告

```text
POST /api/reports/generate
```

请求：

```json
{
  "version_id": 2,
  "format": "markdown"
}
```

响应：

```json
{
  "version_id": 2,
  "report_name": "v1.1_diagnosis_report.md",
  "download_url": "/api/downloads/reports/v1.1_diagnosis_report.md",
  "status": "completed"
}
```

### 6.2 获取报告预览

```text
GET /api/reports/{version_id}/preview
```

## 7. 报告数据来源

| 报告章节 | 数据来源 |
|---|---|
| 基本信息 | `uploaded_file`、`taxonomy_version` |
| 体系统计 | `category_node` 聚合 |
| 结构诊断 | `diagnosis_issue.issue_type` 结构类 |
| 内容诊断 | `diagnosis_issue.issue_type` 内容类 |
| 典型案例 | 高风险和高置信度问题 |
| 维护建议 | `adjustment_suggestion` |
| 版本变更 | `operation_log`、版本 diff |
| 质量评分 | `taxonomy_version.quality_score` |

---

## 8. 异常处理

| 异常 | 处理方式 |
|---|---|
| 版本不存在 | 返回 `VERSION_NOT_FOUND` |
| 节点为空 | 返回 `VERSION_HAS_NO_NODES` |
| 文件写入失败 | 返回 `EXPORT_WRITE_FAILED` |
| 报告目录不存在 | 自动创建 |
| PDF 转换失败 | 保留 Markdown 报告并提示 PDF 失败 |

## 9. 测试设计

### 9.1 单元测试

| 测试项 | 输入 | 期望 |
|---|---|---|
| Excel 行数 | 21090 个节点 | 导出 Excel 有 21090 行数据 |
| 导出字段 | 节点快照 | 包含 6 个标准字段 |
| 报告模板 | 版本数据 | 包含 9 个一级章节 |
| 典型案例筛选 | 多条问题 | 优先选择高风险高置信度问题 |

### 9.2 接口测试

1. 调用版本导出接口。
2. 断言返回下载 URL。
3. 断言导出文件存在。
4. 读取导出 Excel，断言字段完整。
5. 调用报告生成接口。
6. 断言 Markdown 包含真实统计数据和至少 3 类典型问题案例。

---

## 10. 验收标准

1. 可以导出任意版本对应的 Excel。
2. 导出的 Excel 包含标准 6 个字段。
3. 可以生成 Markdown 报告。
4. 报告中包含真实统计数据。
5. 报告中包含至少 3 类典型问题案例。
6. 导出和报告生成不会修改任何版本数据。
