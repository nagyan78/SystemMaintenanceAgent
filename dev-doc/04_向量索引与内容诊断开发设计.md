# 向量索引与内容诊断开发设计

> 功能编号：F04  
> 独立测试目标：为分类节点建立 Qdrant 语义索引，支持相似节点召回，并结合 LLM 识别同义词污染、父子关系异常和疑似重复。  
> 相关源需求：PRD 8.5，技术架构 8、9.2、11.6、15。

---

## 1. 功能目标

将指定版本的节点文本写入 Qdrant，提供相似节点召回能力；对高价值候选问题调用 LLM 做语义判断，生成内容类 `diagnosis_issue`，为后续智能建议生成提供依据。

---

## 2. 功能边界

### 2.1 输入

1. `version_id`
2. `category_node` 节点集合
3. Qdrant 连接配置
4. Embedding 模型配置
5. LLM 模型配置

### 2.2 输出

1. Qdrant Collection：`taxonomy_nodes`
2. 内容诊断问题：
   - `synonym_pollution`
   - `semantic_duplicate`
   - `bad_parent_child_relation`
   - `inconsistent_granularity`
   - `naming_irregular`
3. 相似节点查询结果。

### 2.3 不包含

1. 不执行任何节点修改。
2. 不生成最终维护动作；维护动作属于 F05。
3. Qdrant 或 LLM 不可用时，不影响 F03 结构诊断。

## 4. Qdrant 设计

### 4.1 Collection

```text
taxonomy_nodes
```

### 4.2 Point ID

```text
{version_id}_{category_id}
```

示例：

```text
1_441
```

### 4.3 向量文本模板

```text
节点名称：{category_name}
完整路径：{path_names}
同义词：{syn_list}
层级：{level}
```

### 4.4 Payload

```json
{
  "version_id": 1,
  "category_id": 441,
  "category_name": "苹果",
  "parent_id": 440,
  "level": 5,
  "path_names": "农、林、牧、渔业类产品 > 农业产品 > 水果及坚果 > 水果（园林水果） > 苹果",
  "syn_list": "AirPods, Apple, Apple Music, Apple Pencil, iPhone",
  "is_leaf": true
}
```

## 5. API 设计

### 5.1 建立向量索引

```text
POST /api/diagnosis/content/index
```

请求：

```json
{
  "version_id": 1,
  "rebuild": true
}
```

响应：

```json
{
  "version_id": 1,
  "collection": "taxonomy_nodes",
  "indexed_count": 21090,
  "status": "completed"
}
```

### 5.2 相似节点召回

```text
GET /api/taxonomy/nodes/{node_id}/similar?version_id=1&top_k=10
```

响应：

```json
{
  "version_id": 1,
  "node_id": 441,
  "similar_nodes": [
    {
      "category_id": 442,
      "category_name": "苹果制品",
      "path_names": "食品、饮料、烟、酒类产品 > ...",
      "score": 0.86
    }
  ]
}
```

### 5.3 运行内容诊断

```text
POST /api/diagnosis/content/run
```

请求：

```json
{
  "version_id": 1,
  "sample_limit": 50,
  "top_k": 10
}
```

响应：

```json
{
  "version_id": 1,
  "status": "completed",
  "issue_count": 12,
  "summary": {
    "synonym_pollution": 5,
    "semantic_duplicate": 3,
    "bad_parent_child_relation": 2,
    "inconsistent_granularity": 1,
    "naming_irregular": 1
  }
}
```

## 6. 内容诊断策略

### 6.1 同义词污染

候选选择：

1. `syn_list` 非空。
2. 同义词数量大于 3 的节点优先。
3. 节点路径语义明确的叶子节点优先。

LLM 输入：

```json
{
  "category_id": 441,
  "category_name": "苹果",
  "path_names": "农、林、牧、渔业类产品 > 农业产品 > 水果及坚果 > 水果（园林水果） > 苹果",
  "synonyms": ["AirPods", "Apple", "Apple Music", "Apple Pencil", "iPhone"]
}
```

LLM 输出要求：

```json
{
  "is_issue": true,
  "issue_type": "synonym_pollution",
  "abnormal_synonyms": ["AirPods", "Apple Music", "Apple Pencil", "iPhone"],
  "reason": "节点位于水果路径下，但同义词包含电子品牌和设备名称。",
  "risk_level": "medium",
  "confidence": 0.92
}
```

### 6.2 疑似重复节点

候选选择：

1. 名称完全相同的节点。
2. Qdrant TopK 相似度高于 `0.85` 的节点。
3. 排除自身和同一路径下的父子节点。

LLM 结论：

| 结论 | 后续建议方向 |
|---|---|
| 语义重复 | `merge_node` |
| 多视角合理复用 | `mark_as_valid` |
| 需要人工判断 | 保留内容问题 |

### 6.3 父子关系异常

候选选择：

1. 子节点名称与父节点路径语义差异明显。
2. Qdrant 可召回更相似的候选父节点。
3. 规则诊断中已出现孤立或层级异常的路径优先。

LLM 输出包含：

1. 当前父子关系是否合理。
2. 更合理候选父节点。
3. 判断理由。
4. 置信度。

---

## 7. Prompt 约束

内容诊断 Prompt 必须要求：

1. 只判断是否存在内容问题，不输出修改 SQL。
2. 输出 JSON，不输出额外解释。
3. 置信度范围为 0 到 1。
4. 风险等级只能是 `low`、`medium`、`high`。
5. 不确定时输出 `is_issue=false` 或较低置信度。

---

## 8. 异常处理

| 异常 | 处理方式 |
|---|---|
| Qdrant 连接失败 | 返回 `QDRANT_UNAVAILABLE`，允许继续结构诊断 |
| Embedding 调用失败 | 标记索引任务失败，保留节点数据 |
| LLM 调用失败 | 当前候选标记失败，继续处理下一条 |
| LLM 输出非 JSON | 重试一次；仍失败则记录 `MODEL_OUTPUT_INVALID` |
| 相似节点为空 | 返回空数组，不视为错误 |

## 9. 测试设计

### 9.1 单元测试

| 测试项 | 输入 | 期望 |
|---|---|---|
| 向量文本构造 | 节点对象 | 输出包含名称、路径、同义词、层级 |
| Point ID 构造 | `version_id=1, category_id=441` | `1_441` |
| Payload 构造 | 节点对象 | 包含版本、节点、父节点、路径 |
| 相似节点过滤 | TopK 包含自身 | 结果排除自身 |
| LLM JSON 解析 | 合法 JSON | 转为内容问题对象 |

### 9.2 集成测试

1. 对样例版本建立 Qdrant 索引，断言写入数量为 21090。
2. 查询“苹果”的相似节点，返回 TopK 结果。
3. 使用 Mock LLM 返回同义词污染判断，断言写入 `synonym_pollution` 问题。
4. Qdrant 停止时运行内容诊断，断言接口返回可解释错误，结构诊断数据不受影响。

---

## 10. 验收标准

1. 能为指定版本创建 Qdrant 向量索引。
2. 能召回与目标节点相似的其他节点。
3. 能针对指定节点生成语义分析说明。
4. 能识别至少 5 个同义词异常样例。
5. 能对父子关系异常输出理由和建议方向。
6. Qdrant 或 LLM 失败时不破坏已有版本和结构诊断结果。
