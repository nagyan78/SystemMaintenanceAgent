# 产品标准体系维护

本上下文定义产品分类体系维护工作流中诊断、维护动作和版本产物的业务语言。

## Language

**自动维护执行**:
智能体从同一轮维护建议中筛出符合自动执行门槛且通过规则校验的建议后，自动执行这些建议并生成新版本的原子流程；不符合门槛的建议不会阻塞该执行集合。
_Avoid_: 人工审核后执行、无门槛执行

**维护建议**:
针对已诊断分类体系问题提出的、可校验的分类节点调整提案。
_Avoid_: 审核项、人工决策

**可执行维护动作**:
当前自动维护执行器支持的 `clean_synonym`、`rename_node`、`move_node`、`add_node` 和 `mark_as_valid` 五类调整；其他动作不属于自动执行范围。
_Avoid_: 任意 Agent 建议、未支持动作

**建议校验状态**:
维护建议在自动工作流中的状态序列：`pending` 表示待校验，`validated` 表示已通过全量规则校验并可执行，`executed` 表示已成功应用，`skipped` 表示因自动执行门槛被策略性排除，`failed` 表示校验或执行失败。
_Avoid_: approved、人工批准

**自动执行门槛**:
维护建议只有置信度不低于 `0.80` 且风险等级为 `low` 或 `medium` 时，才具备进入校验与自动执行的资格；高风险或低置信度建议不自动执行，且不会阻塞同轮合格建议。
_Avoid_: need_confirm、人工确认

**新版本**:
由至少一项维护建议成功通过自动维护执行后，从一个基线版本派生出的独立分类体系快照；没有可执行建议时不创建新版本。
_Avoid_: 覆盖原始 Excel、原地修改
