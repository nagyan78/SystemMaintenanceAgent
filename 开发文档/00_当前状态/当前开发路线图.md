# 当前开发路线图

> 状态：CURRENT
> 更新日期：2026-07-17
> 本路线替代旧 M1～M5 和 13～16 作为“当前先做什么”的裁决依据。旧编号仅用于追溯。

## 1. 路线原则

- 优先用户闭环、真实结果和可追溯证据；
- 规则负责确定性诊断，LLM 只做有界语义增强，不直接执行副作用；
- 高风险动作必须经过审核、有效预览、备份和执行门禁；
- 当前课程项目不扩张为分布式多 Agent 平台。

## 2. R1：可信诊断与完整结果——已完成

已交付：

- taxonomy/diagnosis 查询和问题证据主路径；
- “全量轻筛查 → 候选召回 → Agent 深诊断”覆盖漏斗；
- 扫描、命中、候选、深诊断、跳过、失败和未诊断原因；
- 计划范围、问题类型、批次、调用数、Token 和时间预算；
- partial 终态、结果保留、部分完成报告和 workflow/run 证据；
- 固定 Golden Excel 的可重复 precision/recall 验收。

验收结果：`phase5-golden-v1` 连续运行两次，问题 precision/recall 均为 `1.0`。

## 3. R2：安全变更与版本闭环——已完成主验收

已交付：

- 建议结构一致性、稳定问题分类和不完整建议原位再生成；
- 类型/全局批量审核、误报、暂不处理和不可执行项自动忽略；
- 后端能力字段驱动的审核/预览状态机；
- stale preview、空预览和越权动作的后端拒绝；
- 缺失父节点动作去重、执行前备份、事务、幂等和失败证据；
- 新版本、复诊、质量对比、diff、Excel 导出和继续维护入口。

验收结果：真实 1000 行测试文件从 v1.0 生成 v1.1，任务 100%，验证通过；Golden 自动闭环可重复运行。

## 4. R3：报告、导出与演示闭环——已完成主验收

已交付：

- draft/partial/failed/final 四类报告；
- workflow/run、输入/输出版本、问题、建议、审核、执行和验证事实快照；
- Markdown 预览/下载和中文 PDF 下载；
- 固定演示数据、Golden 评价和完整业务脚本。

验收结果：最终 Markdown/PDF 和新版本 Excel 均由真实闭环生成，PDF 已完成渲染检查。

## 5. R4：可靠性与有限智能增强——下一阶段

阶段五主闭环已完成，下一步只做可靠性和规模化增强：

1. 大分类树分页、搜索和执行性能测试；
2. 多人并发审核的乐观锁与冲突处理；
3. 跨进程 runner、共享 checkpoint 与取消/恢复压测；
4. Qdrant 新版本默认重建和外部依赖监控；
5. 扩充业务 Golden set、校准样本和发布门禁；
6. 备份保留策略、容量告警和灾难恢复演练。

已有的终态不变量、run/work item/event 持久化、SSE 续传、有界预算、fallback、审核记忆和 Golden 评价继续保留，不重复建设。

## 6. 暂缓项

- 泛化聊天机器人；
- 六个常驻 Agent 同时协作；
- 分布式 worker/队列；
- 完整模型网关；
- 无明确验收指标的长期记忆；
- 向用户展示原始 chain-of-thought；
- 为展示“智能体感”而增加无业务价值的节点。
