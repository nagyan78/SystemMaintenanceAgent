## 3. 阶段五验收事实

固定 Golden Excel 自动化闭环覆盖：

`上传 → 诊断 → 覆盖/评测 → 证据报告 → 自动审核 → 执行预览 → 去重执行 → 新版本 → 复验 → 最终报告`

- Golden 运行两次均得到同一组 4 个已知问题，precision/recall 均为 `1.0`；
- 执行失败测试确认：失败动作留存 workflow/run/error 证据，且不会创建伪成功版本；
- 全部拒绝或全部不可执行时：任务正常到 100%，生成最终事实报告，不生成空执行预览；
- stale preview 在前端禁用的同时，执行接口也会拒绝；
- 三条问题共用一个缺失父节点时，预览只产生一个父节点补建动作。

真实测试文件 `产品标准体系_test2_1000.xlsx` 的稳定闭环结果：

- 审核批次：`review_eb61508465dc`；
- 工作流：`workflow_784a74bed419`；
- 39 条建议中，8 条具有可靠动作并执行，31 条证据不足的建议自动忽略，无需人工伪造动作；
- 8 条通过建议去重为 3 个实际动作；
- 基线版本 `v1.0`（ID 22）生成 `v1.1`（ID 23），任务进度 100%，验证状态 `passed`；
- 质量评分由 96.5 提升到 100.0，诊断问题由 40 降到 18；剩余问题属于自动忽略的诊断项，不代表未完成流程。

## 4. 当前剩余风险

### P1：大数据量与并发验收

当前主闭环已通过固定样本和 1000 行真实测试文件，但仍需补充更大分类树的分页、搜索、执行耗时和多人并发审核压测。

### P1：跨进程运行时

后台执行和部分 checkpointer 仍以单机进程为边界。多实例部署需要外部 runner、共享 checkpoint 和分布式互斥；这不属于当前课程项目的阶段五完成条件。

### P1：外部依赖

Qdrant、embedding 和 LLM 依赖外部服务。系统已具备预算、fallback 和 partial 降级，但生产使用仍需部署级监控与凭证管理。

## 5. “完成”的统一定义

一个能力只有同时满足以下条件才能标记为完成：

1. 真实 service/repository 实现，不是硬编码或 501；
2. workflow/API 已接入；
3. 前端存在用户可完成的交互；
4. 自动化测试覆盖成功、失败和幂等主路径；
5. 固定样例 Excel 的端到端验收通过；
6. 结果可通过 workflow/run 追溯。
