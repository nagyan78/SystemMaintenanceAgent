## 3. 阶段五验收事实

固定 Golden Excel 自动化闭环覆盖：

`上传 → 诊断 → 覆盖/评测 → 证据报告 → 自动审核 → 执行预览 → 去重执行 → 新版本 → 复验 → 最终报告`

- Golden 运行两次均得到同一组 4 个已知问题，precision/recall 均为 `1.0`；
- 执行失败测试确认：失败动作留存 workflow/run/error 证据，且不会创建伪成功版本；
- 全部拒绝或全部不可执行时：任务正常到 100%，生成最终事实报告，不生成空执行预览；
- stale preview 在前端禁用的同时，执行接口也会拒绝；
- 三条问题共用一个缺失父节点时，预览只产生一个父节点补建动作。

真实测试文件 `产品标准体系_test2_1000.xlsx` 的稳定闭环结果：

- 审核批次：`review_eb61508465dc`；
- 工作流：`workflow_784a74bed419`；
- 39 条建议中，8 条具有可靠动作并执行，31 条证据不足的建议自动忽略，无需人工伪造动作；
- 8 条通过建议去重为 3 个实际动作；
- 基线版本 `v1.0`（ID 22）生成 `v1.1`（ID 23），任务进度 100%，验证状态 `passed`；
- 质量评分由 96.5 提升到 100.0，诊断问题由 40 降到 18；剩余问题属于自动忽略的诊断项，不代表未完成流程。

## 4. 当前剩余风险

### 2026-07-21 提交前规则加固

- 已删除确定性诊断和整改规划中针对具体产品样本名称的业务硬编码；具体回归案例只允许存在于 Golden 测试数据；
- 层级过深改为每条超过七层的根到叶路径记录一个问题，风险按需减少一层、两层、三层及以上分档；
- 节点过宽继续使用八十个直接子节点的固定阈值，风险按最少分组数分档；
- 重名规则只诊断同一父节点下的完全重名，不再把不同路径下的全局同名直接判错；
- 自动工作流只有在独立 AI 明确通过方案后才允许进入确定性校验和执行，复核缺失、不完整或返回 concern 均不得自动执行；
- AI 内容评估按根分类、层级、叶子属性和同义词状态组成联合分层键，固定抽取最多 200 个不重复节点并记录随机种子；合理计 1 分、证据不足计 0.5 分、存在问题计 0 分；
- 综合评分按结构规则 40%、内容规则 10%、AI 内容抽样 50% 计算；AI 抽样未完成时不伪造综合分；
- 过宽整改只接受完整覆盖全部直接子节点的 AI 语义分组，每组 2～80 个节点，新分组 ID 按规范化名称和最小成员 ID 确定性生成；
- 过深整改使用 `collapse_intermediate_node` 删除异常路径上的非根、非叶中间节点，将全部直接子节点改挂到原父节点；重名、宽度恶化、断链或快照预演失败均阻止执行；
- 主工作流不设置人工审核节点，修改方案由独立 AI 复核后自动进入确定性校验、完整快照预演和副本执行；旧人工审核写接口和恢复接口默认返回 410；
- 当前数据域只包含分类树六个 Excel 字段，已移除业务代码、数据库建表、提示词、预演和前端中的外部引用概念；
- 已从 Git 工作树删除两组误跟踪的运行时数据库、WAL/SHM 和测试 XML 备份，共十六个文件、约 110 MB。

上述能力已经进入当前工作树并具备自动化测试；未提交改动仍不等于已发布。外部模型不可用或证据不足时，系统会保留规则结果和失败证据，不会伪造 AI 结论或强行执行结构修改。

## 5. “完成”的统一定义

一个能力只有同时满足以下条件才能标记为完成：

1. 真实 service/repository 实现，不是硬编码或 501；
2. workflow/API 已接入；
3. 前端存在用户可完成的交互；
4. 自动化测试覆盖成功、失败和幂等主路径；
5. 固定样例 Excel 的端到端验收通过；
6. 结果可通过 workflow/run 追溯。
