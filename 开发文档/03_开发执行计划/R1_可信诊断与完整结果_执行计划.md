# R1 可信诊断与完整结果执行计划

> 状态：COMPLETED
> 状态核对：2026-07-17
> 路线来源：[当前开发路线图](../00_当前状态/当前开发路线图.md)

## 目标

让用户上传产品标准体系后，获得覆盖范围明确、可定位、可解释、可追溯的完整问题清单；模型预算耗尽或外部能力不可用时，保留确定性结果并输出部分完成报告。

## 完成范围

1. taxonomy 与 diagnosis 查询 API 支持树浏览、节点搜索、问题筛选和证据查看。
2. 已建立“全量规则轻筛查 → 候选召回 → 重点 Agent 深诊断”的覆盖漏斗。
3. 覆盖 DTO 记录扫描、规则命中、候选、深诊断、问题、跳过、失败和未诊断原因。
4. 诊断计划真实约束节点范围、优先子树、问题类型、抽样、调用数、Token 和 wall time。
5. 预算不足时保留规则结果和已完成 AI 批次，以 `partial` 终态生成 partial report。
6. 问题使用稳定内部代码和中文显示名称，关联节点、路径、证据、置信度、来源和 workflow/run。
7. task、run、问题、建议、审核、执行、版本和报告可通过 evidence API 串联。
8. 失败终态和报告类型不会被后续节点覆盖。

## 非目标

- 不建设分布式多 Agent 平台。
- 不让 LLM 替代确定性规则或直接修改数据。
- 不以增加模型调用量代替覆盖率设计。
