# R3 报告导出与演示闭环执行计划

> 状态：PLANNED  
> 前置条件：R1、R2 验收通过  
> 路线来源：[当前开发路线图](../00_当前状态/当前开发路线图.md)

## 目标

把报告从简单统计摘要升级为一次维护运行的完整审计结果，并形成稳定、可重复的演示闭环。

## 报告必须包含

1. 输入文件、输入版本、输出版本和 workflow run。
2. 扫描覆盖率、候选召回、深诊断范围、模型预算及降级原因。
3. 结构问题、内容问题、中文问题分类和典型证据。
4. 建议、审核决定、编辑内容和拒绝原因。
5. 已执行动作、失败动作、影响范围和回滚结果。
6. 修改前后统计、质量评分、复检结果和评分口径。
7. 新版本 Excel、版本差异和后续维护入口。

## 实施范围

1. 按 workflow run 聚合报告，不只依赖当前版本查询。
2. 统一网页预览、Markdown/PDF 报告与 Excel 导出中的事实口径。
3. 报告清楚区分“完整完成”“部分完成”“失败”，部分完成也必须交付已有结果。
4. 建立固定演示数据、端到端脚本和产物校验。

## 验收标准

- 同一 run 的页面、报告和导出物数据一致。
- 报告中的每个问题、建议和动作均可回溯到证据记录。
- 固定演示数据可重复生成相同结构的报告、新版本 Excel 和版本差异。
- 模型不可用或预算耗尽时，报告仍包含确定性结果、完成范围与明确降级说明。

## 验证

- 后端全量测试、前端 contract test 和 build 全部通过。
- 端到端脚本检查报告字段、文件存在性、版本差异和证据链接。
- 人工检查中文分类、严重程度、表格布局和答辩演示路径。
