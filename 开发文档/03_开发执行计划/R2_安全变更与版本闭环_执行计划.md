# R2 安全变更与版本闭环执行计划

> 状态：PLANNED  
> 前置条件：R1 验收通过  
> 路线来源：[当前开发路线图](../00_当前状态/当前开发路线图.md)

## 目标

让审核通过的建议以可预览、可审计、可回滚、可验证的方式生成新版本，并支持从任意版本继续维护。

## 实施范围

1. 统一 workflow run、输入版本、审核记录、动作和输出版本的证据链。
2. 审核页展示修改前后预览、风险等级、影响节点和校验结果。
3. 补齐产品必要动作；暂不支持的动作必须明确拒绝并说明原因。
4. 动作执行采用事务、幂等键、失败回滚和操作日志。
5. 新版本生成后重新索引、重新诊断，并使用相同口径比较质量。
6. 支持选择任意历史版本发起下一次维护任务。

## 验收标准

- 可从 v1.0 安全生成 v1.1，并追溯每项变化的建议、审核人与执行结果。
- 失败动作不产生半成品版本，重复请求不重复执行。
- v1.1 可完成重新索引、重新诊断和前后质量对比。
- 用户可从 v1.1 或指定历史版本继续创建维护任务。

## 验证

- 后端全量测试覆盖成功、拒绝、失败回滚和幂等路径。
- 前端 contract test 与 production build 通过。
- 固定样例完成“诊断—审核—执行—新版本—复检—续跑”端到端验证。
