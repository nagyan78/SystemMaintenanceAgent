# 阶段四：智能增强 Implementation Plan

> **历史实施计划（SUPERSEDED）**：保留用于追溯四阶段方案，不再决定当前开发顺序。当前路线见 `ROADMAP.md`。

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在可靠的版本闭环、工作单元和动作能力之上，增加滚动规划、模型路由、工具缓存、历史审核记忆、Golden set 外部评估和质量门禁。

**Architecture:** Supervisor 以批次反馈修订 MaintenancePlan；ModelRouter 根据任务类型选择模型档位并受统一预算约束；ToolRegistry 提供权限、缓存和脱敏元数据；AgentMemory 保存可追溯的用户反馈；EvaluationService 使用 Golden set 和运行指标进行独立质量评估，不能由被评估 Agent 自评。

**Tech Stack:** LangChain ChatOpenAI-compatible models、LangGraph、SQLite、Qdrant、Pydantic、FastAPI、Vue 3、pytest。

## 0. 前置条件与边界

前置：阶段一～三全部通过，已具备版本验证、候选 work item、结构化事件和完整动作。

本阶段交付：

- plan 的 focus/range/strategy/budget 全部真实控制执行。
- 每批 10～20 个候选后可扩圈、缩圈或停止。
- planning/diagnosis/suggestion/evaluation 可使用不同模型档位和 fallback。
- 只读工具共享 workflow-scoped cache，副作用工具禁止缓存。
- Agent 使用历史审核反馈，且记忆可追溯、可失效。
- Golden set 输出 precision/recall/F1、动作合法率和危险动作漏拦截率。
- 低质量或低置信度结果进入 triage，不自动进入高风险执行。

本阶段不引入分布式数据库或外部队列；保持本地 SQLite/Qdrant 架构。

- [ ] **Step 3: 建立 cache 表和键**

唯一键由 `workflow_id/version_id/tool_name/normalized_args_hash/data_revision` 组成。结果、created/expires time 和 hit count 持久化。`DataRevisionResolver` 使用以下精确定义：

```python
class DataRevisionResolver:
    TAXONOMY_TOOLS = {"get_node_detail", "get_node_path", "get_children", "get_siblings"}

    def __init__(self, settings: Settings) -> None:
        self.settings = settings

    def for_tool(self, tool_name: str, args: dict[str, Any]) -> str:
        if tool_name in self.TAXONOMY_TOOLS:
            return f"taxonomy:{int(args['version_id'])}"
        if tool_name == "search_similar_nodes":
            version_id = int(args["version_id"])
            version = VersionRepository(self.settings).get_version(version_id)
            return (
                f"qdrant:{self.settings.qdrant_collection}:{version_id}:"
                f"{self.settings.embedding_model}:{version['vector_index_generation']}"
            )
        if "run_id" in args:
            latest_event_id = AgentRunRepository(self.settings).latest_event_id(args["run_id"])
            return f"agent_run:{args['run_id']}:event:{latest_event_id}"
        raise ValueError(f"No data revision strategy for tool {tool_name}")
```

taxonomy version 是不可变完整快照，因此 `taxonomy:{version_id}` 是稳定 revision：同版本查询可复用，新版本 ID 自动失效，不需要为未修改节点计算内容 hash。Qdrant key 额外包含 embedding model 和 `vector_index_generation`；索引 job 每次成功重建/增量提交后原子递增 generation。

同一步给 `taxonomy_version` 增加 `vector_index_generation INTEGER NOT NULL DEFAULT 0`，VersionRepository 提供原子 `increment_vector_index_generation(version_id)`，AgentRunRepository 提供 `latest_event_id(run_id)`；缺少这些 revision 时禁止写 cache，不能退化成固定字符串。
